\subsubsection{Fixtures and Match Occurrences}
\label{sec:org09c9960}
Fixture and match occurrence data is sourced from team Rico in Kambi. The data is shared through the AWS S3 bucket 
\emph{\textbf{prod-datalake-sharing}} which can be updated with more recent data by making a request to Team Rico \footnote{\url{https://jira.services.kambi.com/secure/RapidBoard.jspa?rapidView=1068\&projectKey=DL}}. The
data should be moved to the AWS S3 bucket \emph{\textbf{tzeract-datalake-sharing}} that's owned by Tzeract to ensure that any 
costs associated with this bucket's use is attributed to Tzeract; syncing between S3 buckets can be accomplished by
invoking the following command

\begin{figure}[thp]\centering
  \begin{minted}[
frame=lines,
framesep=2mm,
baselinestretch=1.2,
fontsize=\footnotesize,
linenos
]{bash}
    aws s3 sync s3://prod-datalake-sharing s3://tzeract-datalake-sharing --profile tzeract
  \end{minted}
\caption{S3 Bucket Sync}\label{lst:awsBucketSync}
\end{figure}
\noindent where the \emph{tzeract} profile must be added to the user's AWS config \textasciitilde{}/.aws/config/.
\subsubsection{AWS S3 Bucket Contents}
\label{sec:org41a42bb}
The \emph{tzeract-datalake-sharing} S3 bucket contains fixture, match occurrence and miscellaneous data. Fixture data
may be needed for league, team and player related information; match occurrences will comprise the live events that 
occurred in a given fixture. Note that Betgenius match occurrences belong to Genius fixtures while Betradar match
occurrences belong to Sportradar fixtures; Genius and Sportradar match occurrences are a separate products that
nevertheless share fixtures.
